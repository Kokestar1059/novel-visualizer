# こころ、図解

## ①課題名
こころ、図解 — テキストマイニングで読む夏目漱石『こころ』

## ②課題内容（どんな作品か）
青空文庫の著作権フリー小説（夏目漱石『こころ』）を統計的に解析し、登場人物・場所・主題などの
アクタントとその関係を**ネットワーク図として可視化する研究用ツール**。論文・記事への引用を想定。

- 解析（事実の抽出）は**決定論的手法のみ**（形態素解析・同一文共起・Dice係数）。同じ入力からは必ず同じ図が出る。
- AI（Azure OpenAI）は**自然言語による絞り込みの「クエリ翻訳」だけ**を担い、事実（ノード・関係）は生成させない。
- すべての関係に**原文の根拠（エビデンス＝何文目・原文）**が紐づき、原文へ遡って検証できる。
- ノードは3層＋語り手：人物 / 場所 / アクタント（物・出来事・主題）と、観察者としての語り手「私」。

技術スタック
- 解析（ローカル実行）：Python + GiNZA
- バックエンド：PHP 8 + MySQL/MariaDB（PDO prepared statements）
- フロントエンド：素の HTML / CSS / JavaScript + Cytoscape.js
- AI：Azure OpenAI（自然言語 → クエリJSON 変換のみ）

## ⑤工夫した点・こだわった点
- **「事実」と「解釈」の分離**：統計的に抽出した関係を一次データ（実線）とし、AIには翻訳のみを任せる。AIに本体データを生成・書換させない設計。
- **検証可能性**：全エッジに原文エビデンスを保持し、図の関係から原文へ遡れる。
- **アクターネットワーク的視点**：人物だけでなく、物（手紙・書物・墓 など）・出来事（自殺・卒業 など）・主題（罪・過去・精神・殉死 など）を対等にアクタントとしてノード化。
- **重要アクタントのキュレーション辞書**：頻度では拾えない主題語を人手で選定し、決定論的に採用。
- **語り手「私」を別枠化**：物語全体と共起して無差別ハブになるため、観察者として区別し図の歪みを明示。
- **自然言語での操作**：「先生を中心に2ホップ」「恋の関係だけ見せて」等を AI がクエリJSONへ翻訳。AI には SQL を書かせず、ホワイトリスト方式の prepared statement のみ実行する。
- **UI「墨・紙・藍」**：明朝の見出しと余白の効いたミニマルな配色で、内省的な作品世界に合わせた。
- **セキュリティ**：セッション認証、PDO prepared statements、API は未認証時に 401 JSON。

## ⑥難しかった点・次回トライしたいこと
- **共起の限界**：同一文共起は「関連」は捉えるが、関係の「方向・行為（誰が誰に何をしたか）」を持たない。関係の意味づけには**係り受け解析**が必要 → 次回トライ。
- **頻度 ≠ 重要度**：高頻度の汎用語と低頻度の主題語の選別。品詞の精緻化とキュレーション辞書で対応。
- **2つの世界の連携**：Python 解析（世界A）と PHP 可視化（世界B）を JSON 契約で疎結合に分離しつつ噛み合わせる設計。
- 次回：係り受けによる行為ベースの関係、二次データ（AI解釈）レイヤーの本格化、複数作品対応。
