# specs: 世界A 解析パイプライン（analyze.py）

> 出典・上位文書: [idea.md](../idea.md) §1・§2・§6 / [CLAUDE.md](../CLAUDE.md) §5(ADR)・§6
> Milestone: **M4 解析パイプライン（世界A）**
> ステータス: **仕様（実装前・要承認）**

## 0. 目的とスコープ

青空文庫の日本語小説テキストを **決定論的手法のみ** で解析し、
「アクタント（ノード）」「アクタント間の結びつき（エッジ）」「その原文根拠（エビデンス）」を抽出して、
既存の [import.php](../public/import.php) が取り込める **JSON（nodes.json / edges.json）** を出力する。

- 実行環境: **ローカルPCのみ**（Python + GiNZA）。Sakura には上げない（ADR-001）。
- 入力: `pipeline/source/kokoro.txt`（青空文庫「こころ」／CP932→UTF-8変換済・配置済）。
- 出力: `pipeline/output/nodes.json`, `pipeline/output/edges.json`（既存のダミーを実データで置換）。

### やらないこと（v1スコープ外・将来の別Issue）
- 意味的な関係ラベル（「恋慕」「師事」等）の付与 … **二次データ（AI解釈・破線）** に分離（方針A）。
- `term` の中の object（モノ）/ concept（概念）の細分、媒介者（mediator）の判定等の **ANT的意味づけ** … 二次データ側。
- 複数作品対応の一般化（v1は「こころ」1作で通す。ただしパスは定数化して差し替え可能に）。

## 1. 設計原則（この仕様が守る制約）

- **ADR-002 決定論**: 解析はGiNZAの形態素・固有表現・共起統計のみ。**同入力→同出力**（乱数・LLM不使用）。
- **ADR-003 エビデンス**: すべてのエッジに、結びつきが観測された **原文の文** を紐づける。
- **方針A / Provenance**: 一次データ（＝事実）のみを出す。意味づけ（解釈）は二次データへ。
- **ANT視点**: 人間を特権化せず、人・場所・その他アクタント（`term`）を **対等にノード化** する。
  ただし決定論で取れる粒度に留め、`node_type` は **`person` / `place` / `term` の3層** とする。

## 2. 入力と前処理（決定論）

### 2.1 入力ファイル
- `pipeline/source/kokoro.txt`（UTF-8・CRLF）。`analyze.py` 内で読み込み時に改行を LF に正規化。

### 2.2 除去する青空文庫記法（正規表現・順序に注意）
| 対象 | 例 | 処理 |
|---|---|---|
| ヘッダ（記号説明ブロック） | 先頭〜2本目の `----…----` 行まで | 本文開始前として丸ごと除去 |
| 底本ブロック | `底本：` で始まる行以降・末尾まで | 丸ごと除去 |
| 大/中見出し | `［＃２字下げ］上　先生と私［＃「…」は大見出し］` | **章メタとして分離**し本文からは除去（下記2.4） |
| ルビ | `私《わたくし》` | `《…》` を削除（ルビ内容は捨てる） |
| ルビ開始記号 | `先生一人｜麦藁帽《…》` | `｜` を削除 |
| 入力者注・外字 | `［＃…］` / `※［＃…］` | 削除（外字はv1では捨てる） |
| 行頭字下げ | 全角スペース | 行頭の連続する全角/半角スペースを除去 |

> 除去順序: 底本/ヘッダ切り落とし → 見出し分離 → `※［＃…］`→`［＃…］`→ ルビ`《…》`→`｜`→ 字下げ、の順で行う
> （`※［＃…］` を `［＃…］` より先に処理する）。

### 2.3 文分割とエビデンス基盤
- 前処理後の本文を **句点類（。！？）** で文分割する（改行も文境界の候補）。
- 各文に **1始まりの通し番号 `sentence_id`** を採番する（エビデンスの `sentence_id`）。
- 文の原文文字列を保持（エビデンスの `sentence_text`）。

### 2.4 章メタ（参考情報）
- 大見出し（上・中・下）と中見出し（一・二…）は抽出して章構造として保持できるが、
  **v1のJSON出力には含めない**（nodes/edges/evidence のスキーマに章カラムが無いため）。将来拡張の余地として分離だけしておく。

## 3. ノード抽出（アクタント・決定論・キュレーション辞書方式）

**方式**: person/place/actant の**3つの辞書に載る語だけ**をノードにする（頻度による名詞の自動採用はしない）。
理由: 頻度・品詞では「罪」と「人間」を区別できず、汎用語（人間・力・一種・家…）が紛れ込む。
「物語上の決定的アクタントか」は本質的に解釈なので、**開発時に小説を読んで辞書を人手キュレーション**する
（＝実行時はAI不使用の決定論・原則OK。人物辞書と同じ思想の徹底）。単漢字の誤マッチ（「父」が「叔父」内で拾われる等）は
**longest-match**（長い surface を先にマッチし、その位置を消してから短い語を探す）で防ぐ。

> GiNZA は本方式のノード抽出には**不要**（標準ライブラリのみで実行可能）。将来「アクション/筋」を捉える
> 係り受けエッジを入れる段階で使う想定として `requirements.txt` には残す。

### 3.1 person（人物） — `PERSON_DICT`
- 主要人物＋家族を人間アクターとして列挙: `私 / 先生 / K / 奥さん / お嬢さん / 父 / 母 / 妻 / 兄 / 叔父`。
- **表記ゆれ正規化**: `御嬢さん → お嬢さん`、`お父さん → 父`、`Ｋ → K`（辞書で吸収）。

### 3.2 place（場所） — `PLACE_DICT`
- `鎌倉 / 東京`。

### 3.3 term（その他アクタント） — `ACTANT_DICT`
- 物語上の決定的アクタント/テーマ語だけを厳選（surface一致・頻度非依存）。ANTの非人間アクター（モノ・概念・出来事）。
  - 物: `手紙 / 書物 / 机 / 襖 / 墓 / 電報 / 財産 / 金`
  - 出来事・行為: `恋 / 自殺 / 殉死 / 卒業 / 決心 / 覚悟 / 病気`
  - テーマ・概念: `罪 / 罪悪 / 過去 / 精神 / 明治 / 孤独 / 良心 / 心 / 記憶`
- ★本文に実在する語のみ（ハルシネーション混入なし）。辞書の増減で図の粒度を調整する。

### 3.4 ノードの属性
- `label`: 正規化後の表記。
- `node_type`: `person` / `place` / `term`。
- `frequency`: **総出現回数**（表示・ノードサイズ用）。
- `ref`: 出力時に `n1, n2, …` を採番（nodes.json内のローカル参照。import.php がDB idへ変換）。

## 4. エッジ抽出（アソシエーション・決定論）

### 4.1 手法: 同一文内の共起（co_occurrence）
- **1つの文に同時に出現した2ノード**を「共起」とみなす（無向）。
- 文単位でノード集合を作り、その全ペアを共起としてカウントする。
- `source_ref` / `target_ref` は **ノードid昇順に正規化**（無向グラフの重複を防ぐ）。

### 4.2 重み: Dice係数
- `df(x)` = ノード x が出現する **文数**、`df(a,b)` = a,b が同一文に共起する **文数**。
- `weight = Dice(a,b) = 2 * df(a,b) / (df(a) + df(b))`（0〜1）。DECIMAL(6,4) に収まるよう小数4桁に丸める。
- `edge_type = "共起"`（中立ラベル・方針A）、`method = "co_occurrence"`。

### 4.3 閾値
- 共起文数 `df(a,b) >= COOC_MIN`（採用値 **3**）のペアのみエッジ化（偶発的共起の除去）。

## 5. エビデンス（ADR-003）

- 各エッジについて、a と b が共起した文を **最大 `EVIDENCE_MAX` 件**（初期値 5）記録する。
- 各エビデンス: `sentence_id` / `sentence_text` / `text_span_start` / `text_span_end`。
  - `text_span_*` は、その文中で **先に出現するノード語のマッチ範囲**（文字オフセット）。取得困難な場合は 0 とする。

## 6. 出力JSON（import.php の契約に厳密準拠）

既存 [import.php](../public/import.php) が読む構造をそのまま満たす（フィールド名・ネスト厳守）。

### 6.1 nodes.json
```json
{
  "work": { "title": "こころ", "author": "夏目漱石",
            "source": "青空文庫 https://www.aozora.gr.jp/cards/000148/card773.html" },
  "nodes": [
    { "ref": "n1", "label": "私", "node_type": "person", "frequency": 210 }
  ]
}
```

### 6.2 edges.json
```json
{
  "edges": [
    { "source_ref": "n1", "target_ref": "n2",
      "edge_type": "共起", "weight": 0.6821, "method": "co_occurrence",
      "evidence": [
        { "sentence_id": 12, "text_span_start": 0, "text_span_end": 2,
          "sentence_text": "私はその人を常に先生と呼んでいた。" }
      ] }
  ]
}
```

## 7. 実装構成・依存・実行

```
pipeline/
├── analyze.py            # 本体（前処理→辞書ノード検出→共起→JSON出力）
├── requirements.txt      # ginza, ja_ginza（将来の係り受け用。現方式では未使用）
├── source/
│   └── kokoro.txt        # 入力（UTF-8・配置済）
└── output/
    ├── nodes.json        # 出力（ダミーを実データで置換）
    └── edges.json
```

- 依存: **現方式（辞書ノード検出）は標準ライブラリのみで動作**（GiNZA不要）。
  `requirements.txt` の `ginza`/`ja_ginza` は将来の係り受けエッジ用に残置。
- 実行: `python pipeline/analyze.py`（調整は先頭の3辞書 `PERSON_DICT`/`PLACE_DICT`/`ACTANT_DICT` と `COOC_MIN`/`EVIDENCE_MAX`）。
- **決定論**: 乱数・LLM不使用。同一入力・同一辞書なら常に同一JSON（再現性）。

## 9. 設計判断（確定済み）
- ノード種別: **person / place / term の3層**（object/concept 分別は二次データへ）… 発注者確定。
- ノード抽出: **キュレーション辞書方式**（3辞書に載る語だけ・頻度自動採用なし・longest-match）… 発注者確定。
  頻度ベース（GiNZA名詞＋FREQ_MIN）だと汎用語が混じるため、辞書主義に切替えた。
- 閾値: **COOC_MIN=3 / EVIDENCE_MAX=5**（エッジの偶発共起除去・エビデンス上限）… 発注者選択。
- エッジ手法: **同一文内の共起のみ**（「アクション/筋」を捉える係り受けは将来の別Issue）… 発注者確定。
- 重み: **Dice係数**（0〜1正規化）… 発注者確定。
- 意味ラベル: 一次データは中立ラベル「共起」。「恋慕」等は二次データ（AI解釈・破線）… 方針A・発注者確定。

## 10. 既知の限界・将来
- **キュレーションの限界**: どの語をアクタントとするかは辞書（人手）に依存する＝網羅性は解釈依存。
  辞書に無いアクタントは出ない。逆に恣意性の統制のため、辞書は本文実在を確認し specs に明記する。
- **共起の限界**: エッジは「同じ文に出た」までで、行為・方向を持たない。「誰が何をした」（アクション/筋）は
  係り受け解析が必要 → 将来の別Issueで検討（決定論的に可能。GiNZAはこの時に使う）。
- **意味ラベル・媒介者判定**: 「恋慕」等の関係の意味づけや、ANTの mediator/intermediary 区別は二次データ（AI解釈・破線）。
